    level 01

    Basics

    Navigation and file management — the foundation of everything you do on the command line.

    ls list

    Lists the contents of a directory. Without arguments, shows files in the current working directory.

    syntax
    ls [OPTIONS] [PATH]
    FlagDescription
    -lLong format — shows permissions, owner, size, and date
    -aShow all files, including hidden ones (starting with .)
    -hHuman-readable file sizes (use with -l)
    -tSort by modification time, newest first
    -RRecursive — list subdirectories too
    ls -laLong list including hidden files
    ls -lh /var/logHuman-readable sizes in /var/log
    cd navigate

    Change the current working directory. One of the most fundamental commands — you'll use it constantly.

    syntax
    cd [PATH]
    ShorthandDescription
    cd ~Go to your home directory
    cd ..Go up one directory level
    cd -Switch back to the previous directory
    cd /Go to the root of the filesystem
    cd ~/DocumentsMove to Documents in home folder
    cd ../../etcGo up two levels, then into /etc
    pwd location

    Print Working Directory — outputs the absolute path of your current location in the filesystem.

    syntax
    pwd [OPTIONS]
    FlagDescription
    -LPrint logical path (default), follows symlinks
    -PPrint physical path, resolves symlinks to real path
    pwdOutput: /home/username/projects
    touch create

    Creates an empty file, or updates the access and modification timestamps of an existing file. The easiest way to create a new blank file.

    syntax
    touch [OPTIONS] FILE...
    FlagDescription
    -aUpdate only the access time
    -mUpdate only the modification time
    -tSet a specific timestamp (format: YYYYMMDDhhmm)
    touch notes.txtCreate a new empty file called notes.txt
    touch index.html style.css app.jsCreate multiple files at once
    touch -t 202501010800 report.txtSet file timestamp to Jan 1 2025, 08:00
    mkdir create

    Make Directory — creates one or more new directories at the specified path.

    syntax
    mkdir [OPTIONS] DIRECTORY...
    FlagDescription
    -pCreate parent directories as needed (no error if exists)
    -mSet file mode (permissions) at creation time
    -vVerbose — print a message for each created directory
    mkdir -p projects/linux/labsCreate nested directories at once
    mkdir -m 755 public_htmlCreate with specific permissions
    cp copy

    Copy files or directories from one location to another. The original file is kept — a new copy is made at the destination.

    syntax
    cp [OPTIONS] SOURCE DESTINATION
    FlagDescription
    -rRecursive — required to copy directories
    -iInteractive — prompt before overwriting
    -vVerbose — show each file as it's copied
    -pPreserve timestamps, permissions, and owner
    -uUpdate — only copy if source is newer
    cp report.txt report_backup.txtCopy a file in the same directory
    cp -r myproject/ /backup/myproject/Copy an entire directory recursively
    cp -pv *.conf /etc/myapp/Copy all .conf files, preserving attributes
    mv move / rename

    Move or rename files and directories. Unlike cp, the original is removed — it's cut, not copied.

    syntax
    mv [OPTIONS] SOURCE DESTINATION
    FlagDescription
    -iPrompt before overwriting an existing file
    -vVerbose — print what is being moved
    -nNo-clobber — never overwrite destination
    -uMove only if source is newer than destination
    mv oldname.txt newname.txtRename a file
    mv report.txt ~/Documents/Move file into a different directory
    mv -i *.log /var/archive/Move all logs with overwrite prompt
    rm destructive

    Remove files or directories. Warning: there is no trash can — deleted files are gone permanently.

    syntax
    rm [OPTIONS] FILE...
    FlagDescription
    -rRecursive — required to delete directories
    -fForce — no prompts, ignores nonexistent files
    -iInteractive — prompt before every removal
    -vVerbose — print each file as it's removed
    rm -ri old_project/Remove directory with prompts (safer)
    rm *.logRemove all .log files in current dir
    ! Never run rm -rf / — it deletes your entire root filesystem with no recovery.
    echo output

    Print a line of text to the terminal or redirect it into a file. Commonly used in shell scripts to display messages or write content.

    syntax
    echo [OPTIONS] [STRING...]
    FlagDescription
    -nDo not print the trailing newline
    -eEnable backslash escape sequences (e.g. \n, \t)
    >Redirect output into a file (overwrites)
    >>Append output to a file
    echo "Hello, World!"Print text to terminal
    echo "# My Project" > README.mdCreate a file with content
    echo "new line" >> log.txtAppend to an existing file
    level 02

    Intermediate

    Text processing and search — tools that let you find, filter, and understand data inside files.

    grep search

    Global Regular Expression Print — searches for patterns in files or input streams. Possibly the most powerful text-search tool on Linux.

    syntax
    grep [OPTIONS] PATTERN [FILE...]
    FlagDescription
    -iCase-insensitive search
    -rRecursive search through directories
    -nShow line numbers for each match
    -vInvert match — show lines that do NOT match
    -lList only filenames that contain a match
    -cCount the number of matching lines
    -EExtended regex (same as egrep)
    grep -rn "TODO" ./srcFind all TODO comments in a project
    grep -i "error" /var/log/syslogCase-insensitive error search in logs
    cat app.log | grep -v "DEBUG"Show log lines that aren't DEBUG
    cat read

    Concatenate and print files. Used to view file contents, combine multiple files, or create files from terminal input.

    syntax
    cat [OPTIONS] [FILE...]
    FlagDescription
    -nNumber all output lines
    -bNumber non-blank lines only
    -AShow all special characters (tabs, line endings)
    -sSqueeze consecutive blank lines into one
    cat /etc/os-releaseView system OS information
    cat file1.txt file2.txt > merged.txtCombine two files into one
    find search

    Search for files and directories in a directory hierarchy based on name, type, size, permissions, modification time, and more.

    syntax
    find [PATH] [EXPRESSION]
    ExpressionDescription
    -nameMatch filename (case-sensitive)
    -inameMatch filename (case-insensitive)
    -type fMatch regular files only
    -type dMatch directories only
    -size +10MFiles larger than 10 megabytes
    -mtime -7Modified in the last 7 days
    -execExecute a command on each result
    find . -name "*.sh" -type fFind all shell scripts from here
    find /home -size +100MFind files over 100MB in /home
    find . -name "*.log" -exec rm {} \;Delete all .log files found
    wc count

    Word Count — counts lines, words, and bytes in files. Extremely useful when piped with other commands.

    syntax
    wc [OPTIONS] [FILE...]
    FlagDescription
    -lCount lines only
    -wCount words only
    -cCount bytes
    -mCount characters
    wc -l /etc/passwdCount user accounts in passwd file
    ls /etc | wc -lCount files in /etc using a pipe
    sort text

    Sort lines of text files alphabetically or numerically. Commonly piped with other commands to organise output.

    syntax
    sort [OPTIONS] [FILE...]
    FlagDescription
    -nSort numerically instead of alphabetically
    -rReverse the sort order
    -uUnique — remove duplicate lines
    -k NSort by column number N
    -tSpecify a field delimiter character
    sort names.txtSort names alphabetically
    sort -rn sizes.txtSort numbers from largest to smallest
    cat /etc/passwd | sort -t: -k3 -nSort passwd by UID (column 3)
    head read

    Output the first N lines of a file (default 10). Great for peeking at large files without loading them entirely.

    syntax
    head [OPTIONS] [FILE...]
    FlagDescription
    -n NOutput the first N lines (default: 10)
    -c NOutput the first N bytes
    head -n 5 README.mdShow the first 5 lines of a file
    head -n 20 /var/log/syslogPreview first 20 log lines
    tail read

    Output the last N lines of a file. With -f, it follows the file in real-time — essential for monitoring live logs.

    syntax
    tail [OPTIONS] [FILE...]
    FlagDescription
    -n NShow last N lines (default: 10)
    -fFollow — watch file for new content in real-time
    -FLike -f but retries if file is rotated/renamed
    tail -n 50 /var/log/auth.logLast 50 lines of auth log
    tail -f /var/log/nginx/access.logWatch web server logs live
    |  (pipe) operator

    The pipe operator sends the output of one command as the input to another. It's how you chain Linux tools together to build powerful one-liners.

    syntax
    command1 | command2 | command3
    PatternWhat it does
    cmd | grepFilter output for matching lines
    cmd | sortSort the output
    cmd | wc -lCount the lines of output
    cmd | headShow only the first few lines
    cmd | tee fileShow output AND save it to a file
    ls -la | grep ".sh"List only shell script files
    cat /etc/passwd | cut -d: -f1 | sortList all usernames sorted
    ps aux | grep nginx | wc -lCount nginx processes running
    level 03

    Advanced

    System administration and security — permissions, process monitoring, networking, and automation.

    chmod permissions

    Change file mode bits — controls who can read, write, or execute a file. Uses either symbolic (rwx) or octal (755) notation.

    syntax
    chmod [OPTIONS] MODE FILE...
    ConceptExplanation
    u / g / o / aUser / Group / Others / All
    + / - / =Add / Remove / Set exact permission
    r / w / xRead / Write / Execute bits
    755Owner: rwx, Group: r-x, Others: r-x
    644Owner: rw-, Group: r--, Others: r--
    -RApply recursively to all files/dirs inside
    chmod +x deploy.shMake a script executable
    chmod 644 config.jsonOwner writes, everyone else reads
    chmod -R 755 public/Set permissions on entire directory
    chown permissions

    Change file owner and group. Useful in multi-user environments and when deploying applications that need to run as a specific system user.

    syntax
    chown [OPTIONS] OWNER[:GROUP] FILE...
    FlagDescription
    -RRecursively change ownership
    -vVerbose — show each changed file
    --referenceUse ownership from a reference file
    chown alice report.txtGive alice ownership of report.txt
    chown www-data:www-data /var/wwwSet webserver ownership
    chown -R ubuntu:ubuntu /opt/appRecursively set app ownership
    top monitor

    Real-time process viewer — shows CPU, memory usage, load averages, and all running processes. An interactive dashboard for your system.

    syntax
    top [OPTIONS]
    Key / FlagDescription
    -d NRefresh interval in seconds (default: 3)
    -u USERShow only processes for a specific user
    qQuit top (interactive key)
    kKill a process by PID (interactive key)
    MSort by memory usage (interactive key)
    PSort by CPU usage (interactive key)
    top -d 1 -u www-dataMonitor webserver processes every second
    htopA friendlier alternative (install separately)
    ps processes

    Process Status — shows a snapshot of currently running processes. Unlike top, it's not interactive — you get a one-time list.

    syntax
    ps [OPTIONS]
    FlagDescription
    auxShow all processes for all users (most common)
    -eShow every process
    -fFull-format listing with parent PID
    -u USERShow processes owned by a specific user
    --pid NShow only process with PID N
    ps auxShow all running processes
    ps aux | grep nginxCheck if nginx is running
    ps -ef --forestShow process tree with parent/child relations
    kill destructive

    Send a signal to a process — most commonly used to terminate unresponsive programs by their Process ID (PID).

    syntax
    kill [SIGNAL] PID
    SignalDescription
    -15 (SIGTERM)Graceful shutdown request (default)
    -9 (SIGKILL)Force-kill immediately — cannot be ignored
    -1 (SIGHUP)Hang up — often used to reload config
    -2 (SIGINT)Interrupt — same as pressing Ctrl+C
    kill 1234Gracefully stop process with PID 1234
    kill -9 1234Force-kill a frozen process
    pkill -9 firefoxKill all processes named "firefox"
    ! Always try kill PID (SIGTERM) first. Use kill -9 only when the process won't respond — it skips cleanup and can corrupt data.
    ss network

    Socket Statistics — a faster replacement for netstat. Shows open ports, active connections, and listening services on your machine.

    syntax
    ss [OPTIONS] [FILTER]
    FlagDescription
    -tShow TCP sockets
    -uShow UDP sockets
    -lShow listening sockets only
    -nNumeric — don't resolve hostnames/ports
    -pShow the process name using the socket
    -aShow all (listening + non-listening)
    ss -tlnpSee what's listening and which process owns it
    ss -tn state establishedShow active TCP connections
    ss -u -aAll UDP sockets
    df disk

    Disk Free — reports the amount of disk space used and available on all mounted filesystems.

    syntax
    df [OPTIONS] [FILESYSTEM...]
    FlagDescription
    -hHuman-readable sizes (KB, MB, GB)
    -TShow filesystem type (ext4, tmpfs, etc.)
    -iShow inode usage instead of block usage
    --totalAdd a grand total line at the end
    df -hShow disk usage in human-readable format
    df -hT /homeShow type and space for /home partition
    du disk

    Disk Usage — estimates the space used by files and directories. While df shows whole filesystems, du drills into individual folders.

    syntax
    du [OPTIONS] [PATH...]
    FlagDescription
    -hHuman-readable output
    -sSummary — show only the total for each argument
    -aShow size of all files, not just directories
    --max-depth=NLimit how deep the output goes
    du -sh ~/DownloadsTotal size of your Downloads folder
    du -h --max-depth=1 /varTop-level sizes inside /var
    du -ah . | sort -rh | head -10Top 10 largest files/dirs here
    crontab automation

    Schedule tasks to run automatically at set times using the cron daemon. Each line in a crontab is a scheduled job with a time pattern and a command.

    syntax
    MIN HOUR DAY MON DOW command
    crontab flagDescription
    -eEdit your crontab in the default editor
    -lList your current scheduled jobs
    -rRemove (delete) your crontab entirely
    -u USEROperate on another user's crontab (root only)
    0 2 * * * /opt/backup.shRun backup script every day at 2am
    */15 * * * * /usr/bin/checkhealth.shRun every 15 minutes
    0 9 * * 1 /opt/weekly_report.shEvery Monday at 9am
    ssh network

    Secure Shell — opens an encrypted remote terminal session to another machine. The primary tool for administering Linux servers.

    syntax
    ssh [OPTIONS] user@host
    FlagDescription
    -i FILEUse a specific private key file
    -p PORTConnect on a non-default port (default: 22)
    -LLocal port forwarding (tunnel)
    -vVerbose — useful for debugging connection issues
    -NNo command — just forward ports, don't open shell
    ssh ubuntu@192.168.1.10Connect to a server with password auth
    ssh -i ~/.ssh/mykey.pem ec2-user@hostConnect using an SSH key (AWS style)
    ssh -p 2222 admin@myserver.comConnect on a custom port
    curl network

    Transfer data from or to a server using URLs. Used for making HTTP requests, testing APIs, and downloading files from the command line.

    syntax
    curl [OPTIONS] URL
    FlagDescription
    -o FILESave output to a file
    -OSave with the remote filename
    -X METHODHTTP method: GET, POST, PUT, DELETE
    -HAdd a request header
    -d DATASend POST data
    -sSilent — suppress progress/error output
    -LFollow redirects
    curl https://api.github.comMake a GET request to the GitHub API
    curl -O https://example.com/file.tar.gzDownload a file
    curl -X POST -d "user=admin" http://localhost/loginSend a POST request
